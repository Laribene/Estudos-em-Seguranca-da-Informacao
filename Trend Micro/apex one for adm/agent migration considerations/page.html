Agent Migration Considerations
  • Web Console Move Feature
    • PROS
    • easy move large numbers of agents through the console
    • No third party utility methods required
    • Low bandwidth requirements
    • CONS
    • Cannot move offline/independent agents
    • Cannot be scheduled or set to move when agents check in
  • MSI Install Package (Third Party DeploymentUtility)
    • PROS
    • easy deploy via third party utilities such as SCCM or GPO
    • If the upgrade fails, the endpoint will roll back to the previous version leaving the endpoint protected
    • subject to the parameters of msiexec(logs)
    • CONS
    • Need to engage other teams to deploy the package
    • Badwidth(largura de banda) may be an issue(Agent package é 600 mb)
    • Precisa editar o OfcScan.ini to perform the upgrade
  • IPXfer Utility
    • PROS
    • Lightweight CMD utility
    • Easily scripted
    • CONS
    • Must specify the unload password in the command
    • Large Scale deployment through the Toolbox or third party utility
    • No error return to verify if the agent has moved succesfully
  • Agent Connection Settings
    • PROS
    • quick and easy to configure
    • All endpoints that check in get the updated server setting and migrate
    • CONS
    • Potential for high empact on the Server and Agent level
    • Servidores antigos devem permanecer online até todo os endpoints have checked in
  • Common Unistall Tool (CUT)
    • PROS
    • clean install with no previous agent settings(remove all agents)
    • Very high rate of deployment
    • Easily deplyablethrough third party utility such as SCCM
    • CONS
    • Two- step process which leaves an endpoint vulnerable for a period of time
    • O agente não faz rollback em caso de falha de instalação do agente
    • deve obter CUT from Support, and it expires quarterly
